# 项目名称：LoveAudit (婚恋对象风险评估系统)

## 1. 项目目标
做一个可以在移动端运行的Web 应用，帮助用户评估伴侣的结婚适配度。核心目的是通过“病毒式传播”吸引流量，并通过“隐性心理测试”提供有价值的分析报告，帮助千万男生女生了解自己的对象并进而更了解自己。

## 2. 核心用户体验 (UX)
- **风格**：暗黑、神秘、哥特风。可以有红/黑/粉色配色，配合玫瑰/曼陀罗的抽象线条画。
- **交互**：不应出现超过1条让用户输入的题型。题目分章节，要求尽可能涵盖不同的层次，题目不要超过40个。
- **预留下一个测试的接口**: 如果用户依然有极大的兴趣继续测试，则将更多题目放入第二个测评网站中。可以点击跳转。
- **心理**: 还需考虑用户心理，包括但不限于 耐心有限 隐私考虑 等等

## 3. 功能模块
1.  **Landing Page**：用痛点问题（前任、边界、冷暴力）通过 Hook Model 吸引用户点击。如提出情侣间经常发生矛盾的问题，引起用户的注意和重视。例如：
    你是否因为伴侣向你提起前任而难过?
    你是否被伴侣与其他异性的交往边界困扰？
    你是否处在一段将就的关系中，看不到未来又不想分开？
    你是否经常被NPD 血包 三角测量等网红心理学概念捆绑 越想越害怕 以为找到了原因 精神状态和关系却越来越糟糕？
    …
2.  **测评流程**：
    - 分章节：双方性格、基础安全、三观金钱、矛盾处理、原生家庭
    - 分测试人性格：中间询问用户和用户伴侣是偏理性，偏感性，还是中间，并评估用户和用户伴侣的精神状态，根据他们的性格和精神状态动态调整测试题和评估标准。
    - 形式：选择题为主，题目设计需包含隐形心理标签（安全型、回避型、焦虑型、强势自负、自私、控制欲等）。
3.  **分析页**：模拟 AI 计算过程，增加仪式感。等待过程中，可加入鼓励用户找回自信、或CBT等治疗方法的科普
4.  **结果页**：在用户做完问卷后，不仅要分析用户当前的亲密关系，更要分析用户和其伴侣的精神状态，例如是否处于抑郁、焦虑中，是否被精神控制，是否有反社会倾向，等等，并以此反哺分析用户的亲密关系。但注意不要过度分析，以免加深用户的焦虑抑郁情绪
    - 给出核心结论（神仙伴侣/继续观察/快逃）。
    - 揭开网红名词背后的底层逻辑！！ 
        1. NPD：其实是关系中强势的一方，表现为在冲突中坚持己见不让步，强势，自信。这些人或曾是受害者 并习得这种性格的好处，或由于受教育程度有限固执己见无法辩证思考，或从小被宠坏爱好霸凌他人。不仅限于爱情关系，适用于所有亲密关系，包括家人和朋友之间。
        2. 血包：其实是关系中弱势的一方，表现为在冲突中经常让步，忍耐。是天性善良爱为他人着想的人群，但忘记了过度让步只会让他人得寸进尺。
        3. ...
    - 生成可分享的“人设标签”图片（用于裂变传播）。
    - 提供针对性的相处 Tips（解锁内容）。
